## 四、数据模型冗余分析 (`config/models.yaml`)

来源：`models.yaml` 定义了 **70 个 ORM 模型** + **8 个 Schema**，通过代码生成器自动生成 SQLAlchemy 模型文件。

### 4.1 模型使用情况总览

| 状态 | 数量 | 说明 |
|------|------|------|
| ✅ 活跃使用 | 47+ | 在服务层 + API 路由 + 前端中被引用 |
| ⚠️ 部分使用 | 20+ | 仅在路由或工具层中使用，无独立服务 |
| ❌ **完全冗余** | **2** | 无任何业务代码引用 |
| ❓ 配置缺陷 | 2 | ORM 类型但缺少 table 定义 |

### 4.2 已移除的模型

以下模型及所有相关代码已被移除（后端服务、路由注册、数据库模型、前端页面）：

| 模型 | 涉及范围 | 原因 |
|------|---------|------|
| **`Plugin`** | 15 个服务文件 + 7 个 V1 路由 + V2 路由 + 前端 `AdminPlugins.tsx` + `plugins/` 目录 + `_init_plugins` 启动 | 插件系统不适合个人 AI CMS 场景 |
| **`Theme`** | 模型 + V1 路由 + 前端 `AdminThemeMarketplace.tsx` + `themes/` 目录 | 主题系统已移除，站点样式通过配置管理 |

### 4.3 AI CMS 个人站长场景 — 模型冗余度评估

目标受众：**个人站长**，单用户或少用户运营一个以 AI 能力为核心的内容站点。

## 五、项目方向：AI CMS for 个人站长

### 5.1 项目定位

CardedAI 的核心价值不是 CMS 本身（市场已有 WordPress/Ghost），而是 **AI 原生集成**：
- 情报引擎自动采集行业动态
- 知识引擎实现基于自有内容的 RAG 问答
- 工作流引擎自动化内容流程
- MCP 让 AI Agent 直接管理站点

### 5.2 补齐短板（P1）

| # | 任务 | 涉及 | 优先级 | 状态 |
|---|------|------|--------|------|
